--- title: Public API for the Canada.ca design system guidance ---

The following is a draft. The content is subject to change.

As (a date like 2022-05-17) we are following semantic versioning 2.0.

Audience
Web author and web designer that implement design system component
Major change
Is when the web author need to re-edit their page in order to be conform to the guidance.
Minor change
Is when a new guidance/rule is added but are not conflicting with existing implementation of the component. For example the addition of a new variant for a given component. The author have a new functionality or apply a new rule.
Patch change
Is when there no additional feature. No change are required by the author. Like when we fix a typo in the guidance or re-write an exiting rule to make it clearer to understand or the addition of working example to enforce how the guidance should be applied.
